# Telegram Bot 交互设计详细方案

本方案详细描述 TG Bot 中每个命令的具体交互流程、弹出内容和用户操作方式。

---

## 一、整体用户旅程

### 1. 新用户首次使用流程

```
用户流程:
1. 搜索 Bot → 2. 点击 Start → 3. 收到欢迎消息 → 4. 等待审核 → 5. 审核通过 → 6. 开始使用
```

### 2. 活跃用户日常使用流程

```
用户流程:
1. 接收实时告警 → 2. 查看告警详情 → 3. 发送查询命令 → 4. 获取分析结果 → 5. 调整订阅设置
```

### 3. 管理员管理流程

```
管理员流程:
1. 查看用户列表 → 2. 审核新用户 → 3. 监控系统状态 → 4. 必要时撤销用户
```

## 二、命令详细交互设计

### 1. /start 命令 - 用户注册

#### 触发方式
- 用户在 Telegram 中搜索 Bot 名称
- 点击 "Start" 按钮
- 或直接发送 `/start` 命令

#### 交互流程

**步骤 1：Bot 响应欢迎消息**

```
┌─────────────────────────────────┐
│ 🐋 BTC 鲸鱼订单监控系统  │
│                             │
│ 欢迎来到 BTC 鲸鱼订单监控系统！│
│                             │
│ 您的账号已注册，等待管理员    │
│ 审核通过后即可使用。            │
│                             │
│ 如有问题，请联系管理员。          │
└─────────────────────────────────┘
```

**步骤 2：后台自动操作**

- Bot 创建用户记录
- 设置状态为 `is_active = false`
- 记录注册时间 `created_at`
- 保存用户信息（username, first_name, last_name）

**步骤 3：管理员审核**

- 管理员使用 `/approve <user_id>` 命令
- 系统设置 `is_active = true`
- 用户自动收到激活通知

#### 数据库操作

```sql
-- 插入新用户
INSERT INTO users (
    telegram_id, username, first_name, last_name,
    is_active, is_admin, created_at
) VALUES (?, ?, ?, ?, 0, 0, strftime('%s', 'now'));

-- 激活用户
UPDATE users SET is_active = 1
WHERE telegram_id = ?;
```

#### 消息格式（Markdown）

```markdown
*欢迎来到 BTC 鲸鱼订单监控系统！*

您的账号已注册，等待管理员审核通过后即可使用。

如有问题，请联系管理员。
```

### 3. /subscribe 命令 - 订阅设置

#### 触发方式
- 用户发送 `/subscribe`
- 仅活跃用户可使用

#### 交互流程

**步骤 1：Bot 弹出订阅设置菜单（内联键盘）**

```
┌─────────────────────────────────────┐
│ *订阅设置*                      │
│                                 │
│ 当前订阅的交易所: 全部          │
│ 当前金额阈值: $500,000          │
│                                 │
│ 请选择要订阅的交易所或          │
│ 设置金额阈值:                    │
│                                 │
│ [✅ Binance] [OKX]            │
│ [✅ Bybit] [全部交易所]         │
│                                 │
│ [设置金额阈值] [完成]          │
└─────────────────────────────────────┘
```

**步骤 2：用户点击按钮交互**

- 点击交易所按钮 → 切换订阅状态（✅/❌）
- 点击"全部交易所" → 取消所有交易所限制
- 点击"设置金额阈值" → 进入数字输入模式
- 点击"完成" → 保存设置并显示确认

**步骤 3：设置金额阈值交互**

```
┌─────────────────────────────────────┐
│ *设置金额阈值*                  │
│                                 │
│ 请输入新的金额阈值（美元），     │
│ 例如: 500000                     │
│                                 │
│ 有效范围: $100,000 - $10,000,000 │
│                                 │
│ [发送新阈值] [取消]           │
└─────────────────────────────────────┘
```

**步骤 4：用户输入数字**

- 发送纯数字（如：`1000000`）
- Bot 自动更新阈值
- 发送确认消息

**步骤 5：确认消息**

```
┌─────────────────────────────────────┐
│ ✅ 订阅设置已保存！          │
│                                 │
│ 您将接收：                      │
│ • 交易所: 全部                  │
│ • 金额阈值: >= $1,000,000      │
│                                 │
│ 如需修改，请再次发送          │
│ /subscribe 命令                  │
└─────────────────────────────────────┘
```

#### 数据库操作

```sql
-- 更新订阅的交易所

SET subscribed_exchanges = ?

-- 更新金额阈值

WHERE telegram_id = ?;
```

## 三、用户状态机

### 状态定义

```
┌──────────────┐
│  未注册用户  │
└──────┬───────┘
       │ /start
       ▼
┌──────────────┐
│  待审核用户  │
└──────┬───────┘
       │ /approve
       ▼
┌──────────────┐
│  活跃用户    │
└──────┬───────┘
       │ /revoke
       ▼
┌──────────────┐
│  已撤销用户  │
└──────────────┘
```

### 状态转换规则

| 当前状态 | 触发条件 | 转换到 | 备注 |
|---------|---------|--------|------|
| 未注册 | `/start` | 待审核 | 创建用户记录 |
| 未注册 | `/start` (已存在) | 活跃/待审核 | 根据已有状态 |

| 活跃 | `/revoke` | 已撤销 | 管理员撤销权限 |
| 活跃 | 正常使用 | 活跃 | 更新最后活跃时间 |
| 已撤销 | `/approve` | 活跃 | 管理员恢复权限 |
